# Guardian Custom Rules
#
# The autopilot agent can ADD rules here when it discovers new dangerous patterns.
# Each line is a case-insensitive regex pattern that will block matching Bash commands.
# Lines starting with # are comments.
#
# Format: CATEGORY:::pattern:::reason
# Example: FINANCIAL:::stripe.*charges.*create:::Creating real Stripe charges
#
# RULES:
# - Autopilot can ONLY append to this file, never edit or remove existing rules
# - The built-in guardian patterns (in guardian.sh) cannot be modified by autopilot
# - Each rule makes the system MORE restrictive, never less
#

# Block killing MCP SERVER processes (Node.js), but NOT browser binaries (mcp-chrome)
DESTRUCTIVE:::kill.*playwright-mcp|kill.*playwright.*server:::Killing Playwright MCP server process
DESTRUCTIVE:::pkill.*playwright-mcp|pkill.*playwright.*server:::Killing Playwright MCP server process
DESTRUCTIVE:::killall.*playwright-mcp|killall.*playwright.*server:::Killing Playwright MCP server process
DESTRUCTIVE:::kill.*(mcp-server|mcp_server|mcp.*node):::Killing MCP server process
DESTRUCTIVE:::pkill.*(mcp-server|mcp_server|mcp.*node):::Killing MCP server process
DESTRUCTIVE:::killall.*(mcp-server|mcp_server|mcp.*node):::Killing MCP server process
CREDENTIAL:::echo.*totp-seed|printf.*totp-seed|cat.*totp-seed:::Exposing TOTP seed value
CREDENTIAL:::totp\.sh.*generate.*\|.*(curl|wget|nc):::Piping TOTP code to network
CREDENTIAL:::memory\.db.*(scp|curl):::Exfiltrating memory database
DESTRUCTIVE:::rm.*memory\.db:::Deleting Autopilot memory database
DESTRUCTIVE:::rm.*-rf.*playbooks:::Deleting cached playbooks directory
EVASION:::browser_evaluate.*document\.cookie|browser_run_code.*document\.cookie:::Extracting cookies via MCP browser evaluation
