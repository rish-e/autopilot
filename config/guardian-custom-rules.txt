# Guardian Custom Rules
#
# The autopilot agent can ADD rules here when it discovers new dangerous patterns.
# Each line is a case-insensitive regex pattern that will block matching Bash commands.
# Lines starting with # are comments.
#
# Format: CATEGORY:::pattern:::reason
# Example: FINANCIAL:::stripe.*charges.*create:::Creating real Stripe charges
#
# RULES:
# - Autopilot can ONLY append to this file, never edit or remove existing rules
# - The built-in guardian patterns (in guardian.sh) cannot be modified by autopilot
# - Each rule makes the system MORE restrictive, never less
#

DESTRUCTIVE:::kill.*playwright:::Killing Playwright MCP process — disconnects browser automation permanently
DESTRUCTIVE:::pkill.*playwright:::Killing Playwright MCP process — disconnects browser automation permanently
DESTRUCTIVE:::killall.*playwright:::Killing Playwright MCP process — disconnects browser automation permanently
DESTRUCTIVE:::kill.*mcp:::Killing MCP server process — must be restarted by Claude Code harness
DESTRUCTIVE:::pkill.*mcp:::Killing MCP server process — must be restarted by Claude Code harness
DESTRUCTIVE:::killall.*mcp:::Killing MCP server process — must be restarted by Claude Code harness
CREDENTIAL:::echo.*totp-seed|printf.*totp-seed|cat.*totp-seed:::Exposing TOTP seed value — seeds must stay in keychain
CREDENTIAL:::totp\.sh.*generate.*\|.*(curl|wget|nc):::Piping TOTP code to network — codes are ephemeral
CREDENTIAL:::memory\.db.*(scp|curl):::Exfiltrating memory database
DESTRUCTIVE:::rm.*memory\.db:::Deleting Autopilot memory database
DESTRUCTIVE:::rm.*-rf.*playbooks:::Deleting cached playbooks directory
